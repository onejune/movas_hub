# Dense 特征编码器对比实验

## 实验目的

对比不同 Dense 特征编码方式对 WideDeep 模型效果的影响。

## 实验变量

| 实验 | 编码器 | Dense 维度 | 说明 |
|------|--------|-----------|------|
| baseline | 无 | 0 | 仅使用 sparse 特征 |
| linear_noproject | BatchNorm | 130 | 仅归一化，不降维 |
| minmax | MinMaxScaler | 130 | 归一化到 [0,1] |
| standard | StandardScaler | 130 | Z-score 标准化 |
| log | LogTransform | 130 | log(x+1) 变换 |
| widedeep_dense_v1 | LinearEncoder | 32 | BatchNorm + Linear 降维 |

## 数据配置

## 结论

### 主要发现

1. **整体效果**: Dense 特征对模型 AUC 没有显著提升
   - baseline (无 dense): 0.8254
   - log: 0.8256 (仅提升 0.0002)
   - minmax: 0.8157 (下降 0.0097)

2. **PCOC 校准**: standard 编码器最接近 1 (1.0293)

3. **分业务分析**:
   - mobplus: log 编码提升明显 (+0.028)
   - shein: log 编码提升明显 (+0.016)
   - alibaba: baseline 最优，其他编码器效果下降严重

### 可能原因

1. Dense 特征与 Sparse 特征信息重叠
2. 需要更多特征工程（如交叉特征）
3. Dense 特征本身质量不高
