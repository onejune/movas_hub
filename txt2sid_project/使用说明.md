# SID 生成 Pipeline 使用说明

**版本**: v1.0  
**适用环境**: 阿里云 DSW (CPU/GPU)  
**用途**: 商品文本 → Embedding → RQ-VAE训练 → SID 全流程

---

## 📦 快速开始（3分钟上手）

### 1. 解压文件

```bash
cd /mnt/workspace/MiniOneRec/rq
tar -xzf sid_pipeline_v1.0_latest.tar.gz
cd pipeline
```

### 2. 修改配置

```bash
vi config.sh
```

**必须修改的配置项**：

```bash
# 数据路径（改成你的数据文件）
ITEM_JSON_PATH="/mnt/workspace/MiniOneRec/data/Amazon/index/Toys_and_Games_10k.item.json"

# 模型路径（如果已下载模型，填入路径；否则留空，执行 step2 自动下载）
QWEN_MODEL_PATH="/mnt/workspace/models/Qwen/Qwen2.5-7B"

# RQ-VAE checkpoint（如果已有训练好的模型，填入；否则留空，执行 step4 训练）
RQVAE_CKPT_PATH=""

# 如果已有 embedding 文件，可以直接指定（跳过 step3）
EMB_NPY_PATH=""
```

**设备配置（CPU/GPU切换）**：

```bash
DEVICE_TEXT2EMB="cpu"          # 7B模型需要15GB显存，没有就用cpu
DEVICE_TRAIN_RQVAE="cpu"      # 推荐GPU，cpu也能跑但很慢
DEVICE_GENERATE_SID="cpu"     # 轻量，cpu就够
```

### 3. 执行流程

**完整流程（从零开始）**：

```bash
# 给脚本加执行权限
chmod +x *.sh

# Step1: 安装环境（只需一次）
bash step1_setup.sh

# Step2: 下载模型（只需一次）
bash step2_download_model.sh

# Step3-5: 核心流程
bash step3_text2emb.sh       # 文本 → Embedding
bash step4_train_rqvae.sh    # 训练 RQ-VAE

# ⚠️ 训练完后，把输出的模型路径填入 config.sh 的 RQVAE_CKPT_PATH
vi config.sh

bash step5_generate_sid.sh   # Embedding → SID

# Step7-8: 生成报告
bash step7_generate_summary.sh   # 汇总表格（文本+Embedding+SID）
bash step8_evaluate_quality.sh   # 质量评估
```

**快捷流程（已有模型和数据）**：

```bash
# 如果已有 Qwen 模型、embedding、checkpoint，直接跳到 step5
bash step5_generate_sid.sh
bash step7_generate_summary.sh
bash step8_evaluate_quality.sh
```

## 📂 输出文件说明

所有输出都在 `output/` 目录下：

| 目录 | 文件 | 说明 |
|------|------|------|
| `embeddings/` | `*.emb-qwen-td.npy` | Embedding向量矩阵（N × D） |
| `checkpoints/` | `*/best_collision_model.pth` | 训练好的RQ-VAE模型 |
| `sids/` | `*.index-qwen.json` | 最终SID列表 |
| `summary/` | `*_summary.csv` / `.json` | 汇总表格（Excel可打开） |
| `evaluation/` | `*_quality.json` | 质量评估报告 |

## ⚙️ 常见配置场景

### 场景1：快速测试（用小模型）

```bash
# config.sh 中修改：
QWEN_MODEL_PATH="/mnt/workspace/models/Qwen/Qwen2.5-0.5B"  # 小模型，约1GB
EMB_BATCH_SIZE=16  # 小模型可以加大batch
```

### 场景2：使用已有的 embedding

```bash
# config.sh 中指定：
EMB_NPY_PATH="/mnt/workspace/MiniOneRec/data/Amazon/index/Toys_and_Games_10k.emb-qwen-7b-td-cpu.npy"

# 然后直接跳到 step5
bash step5_generate_sid.sh
```

### 场景3：GPU 加速训练

```bash
# config.sh 中修改：
DEVICE_TRAIN_RQVAE="cuda:0"  # 使用GPU训练
TRAIN_BATCH_SIZE=4096        # GPU可以用更大的batch
```

## 🔧 故障排查

### 问题1：找不到 Qwen 模型

```bash
# 检查模型是否存在
ls /mnt/workspace/models/Qwen/Qwen2.5-7B/

# 如果不存在，执行下载
bash step2_download_model.sh
```

### 问题2：step4 训练后找不到 checkpoint

```bash
# 查看输出目录
ls output/checkpoints/*/

# 找到 best_collision_model.pth，复制完整路径填入 config.sh
vi config.sh
# RQVAE_CKPT_PATH="output/checkpoints/Toys_and_Games_10k/Feb-28-2026_11-30-00/best_collision_model.pth"
```

### 问题3：内存不足

```bash
# 减小 batch size
vi config.sh
# EMB_BATCH_SIZE=4          # 默认8，改成4
# TRAIN_BATCH_SIZE=1024     # 默认2048，改成1024
```

### 问题4：想只处理部分数据测试

```bash
# 手动运行 Python 脚本，加 --num_samples 参数
cd scripts
python text2emb.py \
  --item_json /path/to/item.json \
  --model_path /path/to/qwen \
  --output ../output/embeddings/test.npy \
  --device cpu \
  --num_samples 100  # 只处理前100条
```
